
# WebSocket (Socket.io)

실시간 협업 시스템에서 상태 동기화와 채팅 기능을 구현하기 위해 WebSocket을 학습하고 적용했습니다.

---

## 1. 왜 사용했는가

HTTP 기반 통신은 요청이 있어야만 응답이 가능하기 때문에  
실시간으로 데이터 변경을 반영해야 하는 협업 환경에 적합하지 않았습니다.

칸반 보드에서 작업 상태 변경, 사용자 간 채팅 등의 기능을  
지연 없이 반영하기 위해 WebSocket을 사용했습니다.

---

## 2. 개념 정리

### HTTP vs WebSocket
- HTTP: 요청 → 응답 (단방향 통신)
- WebSocket: 연결 유지 기반 양방향 통신

### Socket.io
- WebSocket 기반 실시간 통신 라이브러리
- 이벤트 기반으로 데이터 송수신 처리

---

## 3. 적용 과정

- 클라이언트와 서버 간 소켓 연결 유지
- 특정 이벤트 발생 시 서버에서 모든 클라이언트로 데이터 전파
- 칸반 보드 상태 변경 시 실시간 동기화 처리

---

## 4. 문제점

- 여러 사용자가 동시에 작업 상태를 수정할 경우 데이터 충돌 발생
- 이벤트가 많아질수록 흐름이 복잡해짐

---

## 5. 해결

- 마지막 수정 기준으로 상태를 업데이트하는 방식으로 충돌 처리
- 이벤트 흐름을 단순화하여 유지보수성과 안정성 개선

---

## 코드 예시

```js
io.on('connection', (socket) => {
  socket.on('updateTask', (data) => {
    io.emit('taskUpdated', data);
  });
});
